
是的，**前端播放必須先取得 NX Token**。完整流程如下：

---

### NX Token 取得時機

`streamStore` 在 **store 初始化時（defineStore 內部，非 action）** 就會自動呼叫：

```ts
// streamStore/index.ts
getNXToken()
  .then((res) => { nxToken.value = res.token })
  .catch(...)
```

→ 對應後端：`GET /v1/auth/nx/getToken`（NxController 用 `nxServerUsername/Password` 向 NX Server 登入，回傳 `{ token, id, ... }`）

---

### Token 的兩個用途

| 用途 | 傳送方式 |
|---|---|
| **m3u8 取得**（`GET /api/v1/auth/nx/{deviceId}/hls`）| 由後端代理，hls.js xhrSetup / fetchSetup 附加 `x-runtime-guid: nxToken` header |
| **所有後續 .ts segment / sub-playlist 請求** | 同上，hls.js 每次請求都帶 `x-runtime-guid` |

---

### 關鍵點

- **後端不自己管理 NX Token**——它只是一個轉發代理，收到 `x-runtime-guid` header 後原樣帶到 NX Server
- **Token 由前端持有**，存在 `streamStore.nxToken`，整個生命週期只在 store 建立時取一次，**沒有 refresh 機制**
- 如果 Token 過期（NX Session 有效期依 `ageS` / `expiresInS` 決定），播放會因 NX Server 401 而失敗
