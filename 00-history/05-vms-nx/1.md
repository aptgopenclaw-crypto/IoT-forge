## NX 串流 API Flow

### 整體架構

兩種模式（live / playback）共用相同的 API 端點 `GET /v1/auth/nx/{deviceId}/hls`，差異只在 query string 的 `pos` 參數。

---

### 即時串流（Live）

```mermaid
sequenceDiagram
    participant Vue as LiveStream/index.vue
    participant Player as NxStreamPlayer.vue
    participant HlsJs as NXStreamPlayer class<br>(useNxStream.ts)
    participant Backend as NxController (Spring Boot)
    participant NxSvr as NX Server

    Vue->>Player: <NxStreamPlayer stream-type="live" device-id="{id}">
    Player->>HlsJs: new NXStreamPlayer(sourceUrl, streamType="live", ...)
    HlsJs->>HlsJs: initHls() → Hls.js instance
    Player->>HlsJs: loadStream()
    HlsJs->>Backend: GET /api/v1/auth/nx/{deviceId}/hls?lo=true<br>Headers: Authorization + x-runtime-guid(nxToken)
    Backend->>NxSvr: (v3) GET {nxServerUrl}/hls/{deviceId}.m3u?lo=true<br>(v4) createSession → fetchAndRewriteM3u8
    NxSvr-->>Backend: master.m3u8 content
    Backend-->>HlsJs: 回傳 application/vnd.apple.mpegurl (m3u8 bytes)
    HlsJs->>HlsJs: Hls.loadSource(m3u8) → 解析 sub-playlist/ts segments
    HlsJs->>Backend: (每個 .ts segment) GET /api/v1/...<br>Headers: Authorization + x-runtime-guid
    Backend->>NxSvr: proxy 轉發
    NxSvr-->>Backend: ts segment
    Backend-->>HlsJs: ts segment
    HlsJs->>Player: videoEle.play()
```

### 歷史播放（Playback）

```mermaid
sequenceDiagram
    participant Vue as Playback/PlaybackStream/index.vue
    participant Player as NxStreamPlayer.vue
    participant HlsJs as NXStreamPlayer class
    participant Backend as NxController

    Vue->>Player: <NxStreamPlayer stream-type="playback"<br>start-time="YYYY-MM-DDTHH:mm:ss"<br>end-time="YYYY-MM-DDTHH:mm:ss">
    Player->>HlsJs: new NXStreamPlayer(..., reqStartPos, reqEndPos)
    Player->>HlsJs: loadStream()
    Note over HlsJs: sourceUrl = BASE_URL/auth/nx/{deviceId}/hls<br>+ ?lo=true&pos={reqStartMs(unix ms)}
    HlsJs->>Backend: GET /api/v1/auth/nx/{deviceId}/hls?lo=true&pos=1751353270750
    Backend->>NxSvr: GET {nxServerUrl}/hls/{deviceId}.m3u?lo=true&pos=1751353270750
    NxSvr-->>Backend: 從指定時間點開始的 m3u8
    Backend-->>HlsJs: m3u8 bytes
    HlsJs->>HlsJs: onFirstFragLoaded → 記錄 curBufferRealStartMs
    HlsJs->>HlsJs: onVideoTimeUpdate → 計算 curRealTimeMs<br>比對 reqEndMs 判斷是否播放結束
```

---

### 關鍵差異整理

| | 即時（Live） | 歷史（Playback） |
|---|---|---|
| **View** | `LiveStream/index.vue` | `Playback/PlaybackStream/index.vue` |
| **stream-type prop** | `"live"` | `"playback"` |
| **pos 參數** | 無 | `?pos={startTime unix ms}` |
| **start/end-time prop** | 無 | `dayjs.unix(startTime).format('YYYY-MM-DDTHH:mm:ss')` |
| **HLS config** | lowLatencyMode | maxBufferLength=240, backBufferLength=60 |
| **timeupdate listener** | 不綁定 | 綁定，監控進度 / 結束點 |
| **結束偵測** | 無 | `curRealTimeMs >= reqEndMs` → stopStream |

---

### NxController 後端邏輯（`/v1/auth/nx/{deviceId}/hls`）

- **v3 (NX 5.x)**：直接呼叫 `nxHttpService.downloadHLS()`，把 NX 回傳的 master m3u8 bytes 直接回應給前端；後續 .ts segments 由前端 hls.js 直連 NX（NX 5.x 不驗證 .ts 層）
- **v4 (NX 6.1+)**：呼叫 `hlsProxyService.createSession()` → `fetchAndRewriteM3u8()`，把 m3u8 內的所有 URL rewrite 成後端代理路徑，前端每個 .ts 請求都經後端轉發並自動加上 `x-runtime-guid`
