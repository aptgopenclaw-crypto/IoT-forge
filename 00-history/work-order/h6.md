## 不同角色的視角

根據你的流程設計，每個角色看到的畫面不同：

| 角色 | 關心什麼 | 主要操作 |
|---|---|---|
| **DEPT_USER**（通報人） | 我通的報單現在到哪了？ | 建立通報、驗證施工結果 |
| **DEPT_ADMIN**（管理者） | 有哪些單還沒審？ | 審核派工、退回 |
| **OPERATOR**（施工者） | 我被派了哪些單？ | 到場打卡、完成維修 |

## 建議的選單結構

```
派工管理（新頂層目錄，從設備管理獨立出來）
│
├─ 📋 所有工單       →  完整的工單列表（含篩選）    所有人可看
│                      可依狀態、通報人、施工者過濾
│
├─ ⏳ 待審核         →  等待 DEPT_ADMIN 派工的單     DEPT_ADMIN
│                      核准 → 自動指派給 OPERATOR
│
├─ 🔧 我的施工任務   →  指派給我的工單                OPERATOR
│                      到場打卡 → 完成維修
│
└─ ✅ 待驗證         →  施工完成等待驗證的單           DEPT_USER
                      核准 → 結案 / 駁回 → 退回施工
```

每個頁面背後都是同一張 `work_orders` 表 + 同一個 `work_order_dispatch` workflow，只是根據**登入角色 + 工單狀態**過濾顯示不同的資料行，以及顯示不同的操作按鈕。

## 完整流程（跨角色、跨頁面）

```
DEPT_USER                    DEPT_ADMIN                 OPERATOR              DEPT_USER
══════════                   ═════════                 ════════              ══════════

[所有工單]                    [待審核]                   [我的施工任務]         [待驗證]
  新增工單                     看到 PENDING 的單           看到 ASSIGNED 的單      看到 REVIEWING 的單
  → 填設備、描述                → 審核內容                   → 到場打卡 (GPS)        → 檢視維修紀錄
  → 送出                        → 核准 / 駁回               → 完成維修              → 核准 / 駁回
                                                             填入 remark, cost
     ↑ workflow start              ↑ step_dispatch           ↑ step_execution         ↑ step_verify
       step_report auto-pass          → step_execution          → step_verify            → step_end
```
