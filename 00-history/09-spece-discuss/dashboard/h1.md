在線率的計算邏輯在 `DeviceService.getStats()` 中，程式碼如下：

```java
// 全部設備中 status = ACTIVE 的數量
long active = all.stream().filter(d -> d.getStatus() == DeviceStatus.ACTIVE).count();

// 全部設備中 lastHeartbeatAt 不為 null 的數量（代表曾經連線過）
long online = all.stream().filter(d -> d.getLastHeartbeatAt() != null).count();

// 在線率 = online / active * 100
onlineRate(active > 0 ? (double) online / active * 100 : 0)
```

### 公式

```
在線率 = 有 lastHeartbeatAt 的設備數 ÷ 狀態為 ACTIVE 的設備數 × 100%
```

### 問題所在

這個計算有兩個潛在問題：

1. **分母是 ACTIVE 設備數**，不是「全部設備數」— 如果大部分設備是 `INACTIVE` 或 `DECOMMISSIONED`，分母會變小，在線率可能被放大
2. **`online` 的判斷只檢查 `lastHeartbeatAt != null`** — 沒有考慮心跳是否過期。一個設備可能三天前有過心跳但已經斷線，仍然被算作「在線」

比較合理的做法應該是加上時間閥值，例如「最後心跳在 5 分鐘內才算在線」：

```java
// 更合理的在線判斷
LocalDateTime threshold = LocalDateTime.now().minusMinutes(5);
long online = all.stream()
    .filter(d -> d.getLastHeartbeatAt() != null && d.getLastHeartbeatAt().isAfter(threshold))
    .count();
```
