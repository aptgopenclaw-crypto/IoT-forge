DIP 的全名是 **Dependency Inversion Principle**（**依賴反轉原則**）。

它是物件導向設計中著名的 **SOLID 原則**裡的其中一個「D」。

其核心精神包含兩句話：
1. **高層模組不應該依賴低層模組，兩者都應該依賴抽象（Abstract / Interface）。**
2. **抽象不應該依賴細節，細節應該依賴抽象。**

在我們前面討論的系統架構中，`audit`（L1）與 `auth`（L2）本來是「低層依賴高層」的關係。為了打破這個限制，我們將介面（Port）抽離到更底層的 `common`（L0）中，讓 L1 和 L2 都去依賴 L0 的這個「抽象介面」，這就是標準的「依賴反轉」實作。
